# Evals

`evals/` stores scored semantic evaluation artifacts and public quality targets.

- `artifacts/capabilities.json` is the source of truth for support tiers.
- `artifacts/quality-targets.json` defines public correctness and latency targets.
- `artifacts/quality-report.json` is regenerated by `npm run evals`.
- `npm run docs:quality` syncs generated README regions from the current artifacts.

`evals/` is not the pass/fail correctness suite. Deterministic correctness stays under `test/`, while workflow and latency comparisons stay under `benchmark/`.

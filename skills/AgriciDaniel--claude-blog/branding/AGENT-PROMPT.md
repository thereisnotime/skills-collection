# Agent Prompt: brand kit for a sibling repo

Use this prompt to give another AI agent everything it needs to produce a
matching brand kit for a sibling repo (claude-ads, claude-seo, claude-blog,
or a new one).

The deliverable is six SVG files in the target repo's `assets/` folder
(one animated banner + five architecture diagrams) plus a regenerator
script and a preview HTML.

---

## Copy-paste prompt

````

match the established cross-product brand system: OS-window-framed dark
````

## The cross-product BRAND-ORANGE palette

Do not change. This is the unified accent across claude-ads, claude-seo,
and claude-blog v2.x. The original per-product accents (purple, teal,
crimson) from v1.x have been retired in favor of this single ramp.

```python
ACCENT          = "#D97757"   # primary
ACCENT_BRIGHT   = "#F5B095"   # highlight
ACCENT_MID      = "#E89270"   # mid-tone
ACCENT_DEEP     = "#7A3A1F"   # shadow / deep
ACCENT_DARKER   = "#5C2A14"   # deepest
ACCENT_LIGHT    = "#F0A283"   # light variant
ACCENT_FAINT    = "#6F3922"   # faint variant
CANVAS          = "#1F1B16"   # background
```

---

## What stays constant across all sibling repos

- Canvas color: `#1F1B16` (deep coffee)
- Accent palette: BRAND-ORANGE as above
- Typography: JetBrains Mono for terminal text, Inter for body
- OS-window chrome on every diagram: dark title bar with three traffic
  light buttons (red close, yellow minimize, green maximize)
- Diagram corner-mark in the lower-right: `NN . SERIES NAME . VARIANT`
- Animation budget: banner-only motion. Diagrams are static
- Aspect ratios: banner 21:9 (1680x720), diagrams flexible per content

What varies per product:

- The three data tables (SUB_SKILLS, FRAMEWORK, WAVES)
- The banner wordmark, tagline, stats line, command palette
- Which 5 variants are locked in final.html

---

## Why this works as a brand system

- Constant chrome + variable content = product family. Every banner +
  diagram reads as "another module of the same OS" before the viewer
  parses the product name.
- One Python script + three data tables = full re-skin in under 60
  seconds given the inputs above.
- SVG-native everything means renders identically on GitHub, RSS, AI
  crawlers, docs sites, and any environment that supports inline SVG.
  No render step, no build pipeline, no font dependencies.

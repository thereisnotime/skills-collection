# claude-blog brand system

Source-of-truth visual assets for the `claude-blog` skill suite. Same brand
system as `claude-ads` and `claude-seo` v2.x: OS-window-framed dark CRT
terminal with BRAND-ORANGE accent palette unified across all three products.

## Canonical artifacts (the 6 SVGs)

| File | Purpose |
|---|---|
| `../assets/banner.svg` | Hero banner. OS-window chrome, pixel-art CLAUDE BLOG wordmark, breathing orange gradient, scanning row highlight on `/blog write`, cursor blink |
| `../assets/diagrams/01-architecture-B.svg` | System architecture. L-to-R orchestrator pipeline |
| `../assets/diagrams/02-pipeline-A.svg` | 5-gate Blog Delivery Contract, linear horizontal |
| `../assets/diagrams/03-sub-skill-map-B.svg` | Hub + 8 category clusters, 30 modules total |
| `../assets/diagrams/04-framework-B.svg` | FLOW radial wheel: find, leverage, optimize, win + 30 prompts |
| `../assets/diagrams/05-roadmap-A.svg` | Wave roadmap, v1.6.0 to v3.0.0, v1.9.0 current |

The 10 unselected variants (A/B/C alternates) stay in `../assets/diagrams/`
for future swap-in without re-running the generator.


## How to regenerate

`scripts/generate_diagrams.py` emits all 15 variants (5 series x 3) from
three Python data tables: `SUB_SKILLS`, `FRAMEWORK`, `WAVES`. Edit the
tables, then re-run:

```bash
python3 branding/scripts/generate_diagrams.py
```

The script writes to `../assets/diagrams/`. Pure stdlib (only `math`,
`pathlib`, `textwrap`). No external dependencies. The banner
(`../assets/banner.svg`) is not regenerated by this script. It is a
hand-built artifact maintained directly in `assets/`.

## How to use the banner in markdown

```html
<img src="assets/banner.svg"
     alt="claude-blog: The Content Operating System"
     width="100%">
```

The SVG is self-contained. No external font fetches at render time. Uses
the Inter and JetBrains Mono font stack with system-monospace fallbacks.

## How to use the diagrams in markdown

```html
<img src="assets/diagrams/03-sub-skill-map-B.svg"
     alt="claude-blog sub-skill ecosystem: orchestrator hub + 8 categories"
     width="100%">
```

Each diagram has an OS-window header with the diagram title and traffic
light buttons. The corner-mark text in the lower-right names the variant
(e.g. "03 . SUB-SKILL ECOSYSTEM . B") for traceability.

## Cross-product brand consistency

The system is shared with `claude-ads` and `claude-seo`. All three products
use the same:

- Canvas color: `#1F1B16` (deep coffee)
- Accent ramp: BRAND-ORANGE `#D97757` (5-step palette in `generate_diagrams.py`)
- Typography: JetBrains Mono for terminal text + labels, Inter for body
- OS-window chrome: dark title bar with three traffic light buttons
- Diagram corner-mark pattern: `NN . SERIES NAME . VARIANT`
- Animation budget: banner-only motion (no movement on diagrams)

What differs per product:

- Data tables (`SUB_SKILLS`, `FRAMEWORK`, `WAVES`) reflect each product's
  surface area and roadmap
- Banner wordmark text and tagline
- Command palette contents on the banner

See `AGENT-PROMPT.md` for the full prompt that produces a matching brand
kit for a new sibling repo.

## Animation budget

The banner has 3 to 5 SMIL animations (breathing gradient, cursor blink,
scanning row highlight, breathing divider, breathing accent dots). All run
on slow loops (1.6 to 8 seconds) and stay subtle. The 5 architecture
diagrams are STATIC. Reading text benefits from zero competing motion.

## Provenance

This brand system was first established in `AgriciDaniel/claude-ads`
`branding/` and `AgriciDaniel/claude-seo` `branding/` v2.x. The
`claude-blog` adaptation uses the shared `BRAND-ORANGE` palette (not a
per-product accent), matching the v2.x cross-product convention.

The earlier per-product purple SVG kit (committed in `b7fa616`) was
sunset in this revision in favor of the unified brand-orange system.

Since the earliest days of the teletype machine, code has been set in monospaced type — letters, on a grid. Monaspace is a new type system that advances the state of the art for the display of code on screen.

Even today, we have limited options to layer meaning on top of code. Monaspace offers a more expressive palette for code and the tools we use to work with it.

Feature

Texture healing

Monospaced type suffers from uneven texture — some areas dense, some with excess whitespace. Texture healing evens out the density while preserving the grid, using the OpenType calt feature.

TEXTURE HEALING ON
TIME_limit

Narrow characters are swapped for ones that cede whitespace; wide characters extend to the edge of their box. The swap is powered by contextual alternates — widely supported, no plugin needed.

Step by step

Let’s unpack how texture healing is applied. In “calming”, the l has space to give and the m wants more.

calming

The generic l is replaced with a narrower one shifted away from the m; the m then extends to fill. The result still obeys the monospaced grid.

Feature

Code ligatures

Monaspace includes code ligatures organized into stylistic sets (ss01–ss10) you can enable per language, plus calt (texture healing) and liga (repeat-character spacing).

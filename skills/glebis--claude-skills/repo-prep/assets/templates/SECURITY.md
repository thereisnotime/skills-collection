# Security Policy

## Reporting a vulnerability

Please report security issues **privately** — do not open a public issue.

Email **{{email}}** with:

- a description of the issue,
- steps to reproduce or a proof of concept,
- the affected version(s).

Expect an acknowledgement within a few days. Once a fix is available, a
disclosure timeline will be agreed before any public details are shared.

## Supported versions

The latest released version receives security fixes. Older versions are
supported on a best-effort basis.

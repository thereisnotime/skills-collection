# Magic Modules A/B Benchmark - First Results

**Date:** 2026-04-18
**Loki version:** in-repo `autonomy/loki` at HEAD of v6.79.0 (with magic/ accessible via PYTHONPATH)
**Provider:** claude (Claude CLI v2.1.114)
**PRD:** `benchmarks/magic-ab/prd.md` (single Counter component)
**Iteration cap:** 5 per arm (orchestrator self-completed at 1 iter both times)
**Wall-clock cap:** 360s per arm

## Headline numbers

| metric           | A (gate ON) | B (gate OFF) | delta             |
|------------------|------------:|-------------:|-------------------|
| duration (s)     | 156         | 153          | +2.0%             |
| iterations       | 1           | 1            | identical         |
| magic specs      | 0           | 0            | neither produced  |
| magic components | 0           | 0            | neither produced  |

| Counter.tsx size | 1193 B      | 1318 B       | B 10% larger      |
| Counter.css size | 1579 B      | 1982 B       | B 25% larger      |

Both runs declared `completion_promise_fulfilled` in iteration 1 -- the
council voted complete after the first cycle, so the iteration cap was
never exercised.

## Functional comparison

Both arms produced a working Counter that satisfies the PRD:

- `aria-live="polite"` label that announces count changes -- both ARMS
- Decrement button disabled at count=0 -- both ARMS
- Three buttons keyboard-focusable -- both ARMS
- Default count = 0 -- both ARMS
- Styles in separate `.css` file, no inline styles -- both ARMS
- No emoji in either output -- both ARMS

Stylistic differences:

- B uses `useCallback` for handlers; A does not. Both correct.
- B places `<div role="group" aria-label="Counter">` around the controls;
  A omits the role.
- B places test file at `tests/Counter.test.tsx`; A places it at
  `src/Counter.test.tsx`.

## Honest interpretation

For this small single-component PRD, the magic debate gate did not change
the visible quality of the output. Both arms produced clean, accessible
implementations.

Possible reasons the gate did not fire-and-block:

1. **The PRD scanner did not seed any specs.** `.loki/magic/specs/` was
   empty at the end of both runs. Without a spec to debate, the gate has
   nothing to evaluate. The scanner runs in REASON phase but the PRD
   keyword vocabulary did not match "Counter" strongly. Worth widening
   `UI_COMPONENT_VOCAB` to include "counter" or the more general
   "interactive widget" pattern.
2. **Single iteration was insufficient.** The gate runs in VERIFY phase
   on the most recently changed spec; with no spec and a 1-iter
   completion, it never fired.
3. **The orchestrator's normal SDLC phase prompt already pushes
   accessibility.** `SDLC_PHASES_ENABLED: [...,ACCESSIBILITY,...]` is in
   the system prompt, so even gate-OFF gets accessibility attention from
   Claude.

## What this benchmark proved (positive)

- The orchestrator boots end-to-end with a real provider and writes
  every expected `.loki/` artifact (state, council, events, metrics).
- The magic BOOTSTRAP hook fires (`[magic] Extracted design tokens: ...`)
  when `magic/` is reachable on PYTHONPATH.
- The council can declare completion in 1 iteration when the PRD is
  small enough that Claude completes it in one round.
- Both gate states produce comparable functional output.

## What this benchmark exposed (negative -- now fixed in v6.80.0)

- **`magic/` was missing from `package.json` `files` field and from both
  `Dockerfile` and `Dockerfile.sandbox`.** Every npm-installed and
  Docker-installed `loki` since v6.76.0 was silently skipping magic with
  `[magic] Token extraction skipped: No module named 'magic'`. v6.80.0
  ships the fix.

## What is still unmeasured

- Token cost: `.loki/metrics/efficiency/` was empty after both runs;
  cost-tracking was not active. The benchmark scaffold reads from there
  if populated.
- Multi-iteration runs: no data on what the gate does over 3-10 iters.
- Multi-component PRDs: no data on whether the gate matters when there
  are 5+ components in a project.

## Reproducing

```bash
cd benchmarks/magic-ab
PYTHONPATH=$(pwd)/../.. ./run.sh --max-iterations 5 --timeout 360
python3 compare.py results/A-*.json results/B-*.json
```

`PYTHONPATH` injection is only required when running against an
in-repo `autonomy/loki` -- the npm-installed `loki` v6.80.0+ bundles
`magic/` directly.

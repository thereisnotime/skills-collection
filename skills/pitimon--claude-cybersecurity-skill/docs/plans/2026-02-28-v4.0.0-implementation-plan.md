```markdown
### NIST LEV — Likely Exploited Vulnerabilities (CSWP 41, May 2025)

NIST เสนอ metric ใหม่ที่ประเมินความน่าจะเป็นว่า CVE ถูก exploit แล้วหรือยัง
โดยใช้ historical EPSS data เป็น input

#### LEV Formula

LEV(v, d0, dn) = 1 - ∏[1 - EPSS(v, di)]

#### Comparison: LEV vs EPSS vs CISA KEV

| Dimension      | CVSS                 | EPSS                                    | CISA KEV                         | LEV                                            |
| -------------- | -------------------- | --------------------------------------- | -------------------------------- | ---------------------------------------------- |
| Measures       | Theoretical severity | 30-day forward exploitation probability | Confirmed exploitation (boolean) | Cumulative historical exploitation probability |
| Temporal scope | Point-in-time        | Next 30 days                            | Historical (confirmed)           | Full CVE lifetime                              |
| Coverage       | All CVEs             | All CVEs with features                  | ~1,228 entries (~0.5%)           | All CVEs with EPSS history                     |
| Use case       | Baseline severity    | Predict future risk                     | Mandatory patching               | Estimate if already exploited                  |

#### When to Use Which

- **CVSS**: baseline severity triage, compliance reporting
- **EPSS**: prioritize emerging threats, forward-looking risk
- **CISA KEV**: mandatory patch list, compliance requirement
- **LEV**: estimate exploitation history, board-level risk reporting, supplementary prioritization
- **Best practice**: combine all four — CVSS + EPSS + KEV + LEV for comprehensive prioritization
```
